015/**
016 * A simple format parser for String-based configuration or data files where JSON is overkill.
017 * Supports only one type, String, but allows each String to have arbitrary nested levels of
018 * String children as if in sub-lists. You can interpret the Strings however you want, and
019 * quoting each String isn't necessary if they are just one word ("bare words" are allowed).
020 * <br>
021 * The main way of using this is to get an ObText.ItemIterator value using {@link #iterator()},
022 * which acts as a normal Iterator over the top-level Strings (not children of anything), but
023 * to call its {@link ItemIterator#hasChild()} method when you expect potential child elements,
024 * then {@link ItemIterator#children()} to get another ItemIterator over the child elements if
025 * you want to explore deeper.
026 * <br>
027 * This implements Collection of String but is (mostly) unmodifiable; you can call
028 * {@link #parse(CharSequence)} to append the results of parsing more formatted text, or call
029 * {@link #clear()} to remove all data. {@link #add(Object)} and {@link #remove(Object)} are
030 * not implemented and throw exceptions. A quirk of how this implements Collection is that it
031 * only considers the top-level Strings to be part of the Collection for length and for
032 * {@link #contains(Object)}, and will ignore child strings unless you access them via
033 * {@link ItemIterator#children()} on an item that has at least one child.
034 * <br>
035 * Format example:
036 * <pre>
037 * hello world
038 * 'how are you today?' [just great thanks]
039 * hooray!
040 *
041 * complexity?
042 * [it is possible [yes this is a good example]
043 * 'escapes like \[\'\] all work'
044 * ]
045 *
046 * comments are allowed // like this
047 * comments can have different forms # like this
048 * // block comments like in c are allowed
049 * / * but because this example is in javadoc, this example is not actually a comment * /
050 * // remove the spaces between each slash and asterisk to make the last line a comment.
051 * /[delimit/or block comments with delimiters/delimit]/
052 *
053 * '''
054 * raw strings (heredocs) look like this normally.
055 *     they permit characters without escapes, ]][][[ \/\/\ ,
056 *     except for triple quotes.
057 *     they keep newlines and indentation intact,
058 * except for up to one newline ignored adjacent to each triple quote.
059 * '''
060 *
061 * [[different[
062 * if you may need triple quotes
063 *     in the raw string, use a different syntax that allows delimiters.
064 * here, the delimiter is '''different''', just to be different.]different]]
065 * </pre>
066 * <br>
067 * Inspired strongly by STOB, http://stobml.org/ , but no code is shared and the format is
068 * slightly different. The main differences are that ObText supports nested block comments
069 * using the syntax {@code /[delimiter/contents/delimiter]/} where delimiter may be empty
070 * but must match on both sides, and contents is the body of the comment. ObText uses Python-
071 * like "heredoc" syntax for raw strings surrounded by triple-apostrophes '''like so''' with
072 * optional initial and final newlines in the raw string ignored. An alternate raw string
073 * syntax is present that allows delimiters, using the syntax
074 * {@code [[delimiter[contents]delimiter]]}, where again delimiter may be empty and contents
075 * is the body of the raw string. We use square brackets in place of STOB's curly braces to
076 * mark children associated with a string.
077 */
078@Beta

126    /**
127     * Parses the given text (a String or other CharSequence) and appends it into this ObText.
128     * @param text a CharSequence (such as a String) using ObText formatting, as described in this class' JavaDocs
129     * @return this ObText object after appending the parsed text, for chaining
130     */
131    public ObText parse(CharSequence text)
132    {
133        m.setTarget(text);
134        nesting.clear();
135        int t = -1;
136        while (m.find()) {
137            if (m.isCaptured(stringId)) {
138                strings.add(m.group(stringId));
139                neighbors.add(1);
140                if(nesting.isEmpty()) length++;
141            }
142            else if(m.isCaptured(openId))
143            {
144                nesting.add(neighbors.size - 1);
145            }
146            else if(m.isCaptured(closeId))
147            {
148                neighbors.incr(t = nesting.pop(), neighbors.size - t - 1);
149                if(t < neighbors.size - 1)
150                    neighbors.set(neighbors.size-1, 0);
151            }
152        }
153        return this;
154    }

178    /**
179     * A one-way iterator through this ObText's String items. Note that this can get
180     * an iterator into a child sequence with {@link #children()}, which should
181     * only be called after checking that a child exists with {@link #hasChild()}.
182     * {@link #remove()} is not supported, but {@link #next()} and {@link #hasNext()}
183     * are, of course.
184     */
185    public class ItemIterator implements Iterator<String>
186    {
187        ItemIterator()
188        {
189        }
190        ItemIterator(int i)
191        {
192            current = i % neighbors.size;
193            index = current;
194        }
195        int index = 0, current = -1;
196        /**
197         * Returns {@code true} if the iteration has more elements.
198         * (In other words, returns {@code true} if {@link #next} would
199         * return an element rather than throwing an exception.)
200         *
201         * @return {@code true} if the iteration has more elements
202         */
203        @Override
204        public boolean hasNext() {
205            return index < neighbors.size && (current < 0 || neighbors.get(current) > 0);
206        }
207        /**
208         * Returns {@code true} if the ObText.Item has any child elements.
209         * (In other words, returns {@code true} if {@link #children} would
210         * return an ItemIterator rather than throwing an exception.)
211         *
212         * @return {@code true} if this has any children
213         */
214        public boolean hasChild() {
215            return index < neighbors.size - 1 && ((current < 0 && neighbors.get(0) > 1) || neighbors.get(current) > 1);
216        }
217
218        /**
219         * Returns the next element in the iteration.
220         *
221         * @return the next element in the iteration
222         * @throws java.util.NoSuchElementException if the iteration has no more elements
223         */
224        @Override
225        public String next() {
226            if(current < 0)
227            {
228                if(strings.isEmpty())
229                    throw new java.util.NoSuchElementException("No more sibling items in ObText object");
230                current = 0;
231                index = neighbors.get(0);
232                return strings.get(0);
233            }
234            if(index >= neighbors.size || neighbors.get(current) <= 0)
235                throw new java.util.NoSuchElementException("No more sibling items in ObText object");
236            index = neighbors.get(index) + (current = index);
237            return strings.get(current);
238        }
239        /**
240         * Returns the first child of this ObText.Item and descends into the sequence of child elements.
241         *
242         * @return the first child of this item
243         * @throws java.util.NoSuchElementException if the iteration has no children
244         */
245        public ItemIterator children() {
246            if(current < 0)
247            {
248                if(neighbors.size <= 0 || neighbors.get(0) == 1)
249                    throw new java.util.NoSuchElementException("No current children in ObText object");
250                return new ItemIterator(1);
251            }
252            if(current >= neighbors.size - 2 || neighbors.get(current) == 1)
253                throw new java.util.NoSuchElementException("No current children in ObText object");
254            return new ItemIterator(current+1);
255        }
256
257        @Override
258        public void remove() {
259            throw new UnsupportedOperationException("remove() not supported");
260
261        }
262    }

264    // Used to generate randomized delimiters using up to 9 non-English letters.
265    // call while assigning your state with randomChars(state += 0x9E3779B97F4A7C15L, myChars)
266    // that assumes you have a 9-element char[] called myChars
267    // as long as z/state is deterministic (i.e. based on a hash), this should be too
268    private static void randomChars(long z, char[] mut)
269    {
270        z = (z ^ (z >>> 30)) * 0xBF58476D1CE4E5B9L;
271        z = (z ^ (z >>> 27)) * 0x94D049BB133111EBL;
272        z ^= (z >>> 31);
273
274        mut[0] = letters[(int)(128 + (z & 127))];
275        mut[1] = letters[(int)(128 + (z >>> 7 & 127))];
276        mut[2] = letters[(int)(128 + (z >>> 14 & 127))];
277        mut[3] = letters[(int)(128 + (z >>> 21 & 127))];
278        mut[4] = letters[(int)(128 + (z >>> 28 & 127))];
279        mut[5] = letters[(int)(128 + (z >>> 35 & 127))];
280        mut[6] = letters[(int)(128 + (z >>> 42 & 127))];
281        mut[7] = letters[(int)(128 + (z >>> 49 & 127))];
282        mut[8] = letters[(int)(128 + (z >>> 56 & 127))];
283    }

294    protected static void appendQuoted(StringBuilder sb, String text, Matcher bareFinder)
295    {
296        if(text == null || text.isEmpty()) {
297            sb.append("''");
298            return;
299        }
300        bareFinder.setTarget(text);
301        if(!bareFinder.find())
302            sb.append(text);
303        else
304        {
305            raw.setTarget(text);
306            if(raw.find()) {
307
308                if (text.contains("'''")) {
309                    long state = CrossHash.Wisp.hash64(text);
310                    char[] myChars = new char[9];
311                    int count;
312                    do {
313                        randomChars(state += 0x9E3779B97F4A7C15L, myChars);
314                        count = StringKit.containsPart(text, myChars, "]", "]]");
315                    } while (count == 12);
316                    sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
317                            .append(myChars, 0, count).append("]]");
318                } else {
319                    sb.append("'''\n").append(text).append("\n'''");
320                }
321            }
322            else if(!text.contains("'"))
323            {
324                sb.append('\'').append(text).append('\'');
325            }
326            else
327            {
328                if(text.contains("\""))
329                {
330                    if(text.contains("'''"))
331                    {
332                        long state = CrossHash.Wisp.hash64(text);
333                        char[] myChars = new char[9];
334                        int count;
335                        do
336                        {
337                            randomChars(state += 0x9E3779B97F4A7C15L, myChars);
338                            count = StringKit.containsPart(text, myChars);
339                        }while(count == 9);
340                        sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
341                                .append(myChars, 0, count).append("]]");
342                    }
343                    else
344                    {
345                        sb.append("'''\n").append(text).append("\n'''");
346                    }
347                }
348                else
349                {
350                    sb.append('"').append(text).append('"');
351                }
352            }
353        }
354    }

384    /**
385     * Deserializes an ObText that was serialized by {@link #serializeToString()} or {@link #toString()}, and will
386     * ignore the prefix and suffix that toString appends for readability (these are "ObText object: [[[[ " and " ]]]]",
387     * for reference). This is otherwise the same as calling the constructor {@link #ObText(CharSequence)}.
388     * @param data a String that is usually produced by serializeToString or toString on an ObText
389     * @return a new ObText produced by parsing data (disregarding any prefix or suffix from toString() )
390     */
391    public static ObText deserializeFromString(String data)
392    {
393        if(data.startsWith("ObText object: [[[[\n"))
394        {
395            return new ObText(data.substring(20, data.length() - 5));
396        }
397        return new ObText(data);
398    }

400    private static void iterate(StringBuilder sb, ObText.ItemIterator it)
401    {
402        while (it.hasNext()) {
403            appendQuotedObText(sb, it.next());
404            sb.append('\n');
405            if (it.hasChild()) {
406                sb.append("[\n");
407                iterate(sb, it.children());
408                sb.append("]\n");
409            }
410        }
411    }

414     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries.
415     * This returns a {@link ContentMatcher} object that you must call setTarget on before using it.
416     * The argument(s) to setTarget should be the text that might contain quotes, heredoc-style quotes, or just bare
417     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
418     * left or returning true and advancing the search if a String was found. The String might be a special term in some
419     * cases, like "[" and "]" without quotes being syntax in ObText that don't contain usable Strings. That's why,
420     * after a String was found with find(), you should check {@link ContentMatcher#hasMatch()} to verify that a match
421     * was successful, and if that's true, then you can call {@link ContentMatcher#getMatch()} to get the un-quoted
422     * contents of the next String in the target.
423     * @return a {@link ContentMatcher} that must have one of its setTarget() methods called before it can be used
424     */
