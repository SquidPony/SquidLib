012/**
013 * An alternative to AStarSearch when you want to fully explore a search space, or when you want a gradient floodfill.
014 * It's currently significantly faster that AStarSearch, and also supports pathfinding to the nearest of multiple
015 * goals, which is not possible with AStarSearch. This last feature enables a whole host of others, like pathfinding
016 * for creatures that can attack targets between a specified minimum and maximum distance, and there's also the
017 * standard uses of Dijkstra Maps such as finding ideal paths to run away. One unique optimization made possible by
018 * Dijkstra Maps is for when only one endpoint of a path can change in some section of a game, such as when you want to
019 * draw a path from the player's current cell to the cell the mouse is over, and while the mouse can move quickly, the
020 * player doesn't move until instructed to. This can be done very efficiently by setting the player as a goal with
021 * {@link #setGoal(Coord)}, scanning the map to find distances with {@link #scan(Collection)}, and then as long as the
022 * player's position is unchanged (and no obstacles are added/moved), you can get the path by calling
023 * {@link #findPathPreScanned(Coord)} and giving it the mouse position as a Coord. If various parts of the path can
024 * change instead of just one (such as other NPCs moving around), then you should set a goal or goals and call
025 * {@link #findPath(int, Collection, Collection, Coord, Coord...)}. The parameters for this are used in various methods
026 * in this class with only slight differences: length is the length of path that can be moved "in one go," so 1 for most
027 * roguelikes and more for most strategy games, impassable used for enemies and solid moving obstacles, onlyPassable can
028 * be null in most roguelikes but in strategy games should contain ally positions that can be moved through as long as
029 * no one stops in them, start is the NPC's starting position, and targets is an array or vararg of Coord that the NPC
030 * should pathfind toward (it could be just one Coord, with or without explicitly putting it in an array, or it could be
031 * more and the NPC will pick the closest).
032 * <br>
033 * As a bit of introduction, the article http://www.roguebasin.com/index.php?title=Dijkstra_Maps_Visualized can
034 * provide some useful information on how these work and how to visualize the information they can produce, while
035 * http://www.roguebasin.com/index.php?title=The_Incredible_Power_of_Dijkstra_Maps is an inspiring list of the
036 * various features Dijkstra Maps can enable.
037 * <br>
038 * If you can't remember how to spell this, just remember: Does It Just Know Stuff? That's Really Awesome!
039 * Created by Tommy Ettinger on 4/4/2015.
040 */

045    /**
046     * The type of heuristic to use.
047     */
048    public enum Measurement {
049
050        /**
051         * The distance it takes when only the four primary directions can be
052         * moved in. The default.
053         */
054        MANHATTAN,
055        /**
056         * The distance it takes when diagonal movement costs the same as
057         * cardinal movement.
058         */
059        CHEBYSHEV,
060        /**
061         * The distance it takes as the crow flies. This will NOT affect movement cost when calculating a path,
062         * only the preferred squares to travel to (resulting in drastically more reasonable-looking paths).
063         */
064        EUCLIDEAN;
065
066        public double heuristic(Direction target) {
067            switch (this) {
068                case CHEBYSHEV:
069                    return 1.0;
070                case EUCLIDEAN:
071                    switch (target) {
072                        case DOWN_LEFT:
073                        case DOWN_RIGHT:
074                        case UP_LEFT:
075                        case UP_RIGHT:
076                            return root2;
077                        default:
078                            return 1.0;
079                    }
080            }
081            return 1.0;
082        }
083
084        public int directionCount() {
085            switch (this) {
086                case MANHATTAN: return 4;
087                default: return 8;
088            }
089        }
090    }

294    /**
295     * Used to initialize or re-initialize a DijkstraMap that needs a new physicalMap because it either wasn't given
296     * one when it was constructed, or because the contents of the terrain have changed permanently (not if a
297     * creature moved; for that you pass the positions of creatures that block paths to scan() or findPath() ).
298     *
299     * @param level a 2D double array that should be used as the physicalMap for this DijkstraMap
300     * @return this for chaining
301     */
302    public DijkstraMap initialize(final double[][] level) {
303        width = level.length;
304        height = level[0].length;
305        gradientMap = new double[width][height];
306        physicalMap = new double[width][height];
307        costMap = new double[width][height];
308        targetMap = new Coord[width][height];
309        for (int x = 0; x < width; x++) {
310            System.arraycopy(level[x], 0, gradientMap[x], 0, height);
311            System.arraycopy(level[x], 0, physicalMap[x], 0, height);
312            Arrays.fill(costMap[x], 1.0);
313        }
314        standardCosts = true;
315        initialized = true;
316        return this;
317    }

576     * @param x
577     * @param y
578     */
579    public void setGoal(int x, int y) {
580        if (!initialized || x < 0 || x >= width || y < 0 || y >= height) return;
581        if (physicalMap[x][y] > FLOOR) {
582            return;
583        }

585        goals.add(encode(x, y));
586        gradientMap[x][y] = 0.0;

2688     * Scans the dungeon using DijkstraMap.scan with the listed fearSources and start point, and returns a list
2689     * of Coord positions (using Manhattan distance) needed to get further from the closest fearSources, meant
2690     * for running away. The maximum length of the returned list is given by length; if moving the full
2691     * length of the list would place the mover in a position shared by one of the positions in onlyPassable
2692     * (which is typically filled with friendly units that can be passed through in multi-tile-
2693     * movement scenarios), it will recalculate a move so that it does not pass into that cell.
2694     * The keys in impassable should be the positions of enemies and obstacles that cannot be moved
2695     * through, and will be ignored if there is a fearSource overlapping one. The preferLongerPaths parameter
2696     * is meant to be tweaked and adjusted; higher values should make creatures prefer to escape out of
2697     * doorways instead of hiding in the closest corner, and a value of 1.2 should be typical for many maps.
2698     * The parameters size, preferLongerPaths, impassable, and the varargs used for fearSources will be cached, and
2699     * any subsequent calls that use the same values as the last values passed will avoid recalculating
2700     * unnecessary scans. Calls to findFleePath will cache as if size is 1, and may share a cache with this function.
2701     * The parameter size refers to the side length of a square unit, such as 2 for a 2x2 unit. The
2702     * parameter start must refer to the minimum-x, minimum-y cell of that unit if size is &gt; 1, and
2703     * all positions in the returned path will refer to movement of the minimum-x, minimum-y cell.
2704     * <br>
2705     * This caches its result in a member field, path, which can be fetched after finding a path and will change with
2706     * each call to a pathfinding method.

2708     * @param size              the side length of the creature trying the find a path
2709     * @param length            the length of the path to calculate
2710     * @param preferLongerPaths Set this to 1.2 if you aren't sure; it will probably need tweaking for different maps.
2711     * @param impassable        a Set of impassable Coord positions that may change (not constant like walls); can be null
2712     * @param onlyPassable      a Set of Coord positions that this pathfinder cannot end a path occupying (typically allies); can be null
2713     * @param start             the start of the path, should correspond to the minimum-x, minimum-y position of the pathfinder
2714     * @param fearSources       a vararg or array of Coord positions to run away from

2716     */

2735                && Arrays.equals(fearSources, cachedFearSources)) {
